# Charte Fondatrice — Le Maître Mot

## Positionnement
Le projet vise un système pédagogique déterministe, explicable et industrialisable,
destiné à des élèves, enseignants et administrateurs non techniques.

La réduction de l'ambiguïté est prioritaire sur toute autre considération.

## Règles fondatrices
- Déterminisme avant élégance.
- Source de vérité unique et explicite.
- Séparation stricte produit / technique / UX.
- Toute décision structurante doit être visible dans le modèle ou l'UI.
- Lisibilité admin prioritaire sur toute optimisation locale.
- Priorisation stricte P0 / P1 / P2.

## Refus systémiques
- Magie, heuristiques implicites, fallbacks silencieux.
- Refactor global non demandé.
- Mélange bugfix / redesign produit.
- États système à interprétations multiples.

## Risques majeurs
- Chapitre visible mais non fonctionnel.
- Admin ne comprenant pas ce qu'il configure.
- Dette technique invisible.
- Comportement changeant sans raison visible.
